What is the use of read me in github

A README on GitHub is the main introduction page of your project. It helps people quickly understand what your project is, what it does, and how to use it.

For your Power BI dashboard project, a README can include:

Project title

Short description of the dashboard

Features/KPIs used

Dataset information

Screenshot of the dashboard

Tools used (Power BI, Excel, etc.)

Insights from the analysis

How to open/use the file

It is important because:

Makes your project look professional

Helps recruiters understand your work quickly

Increases chances of getting noticed on GitHub and LinkedIn

Explains your project without opening the file

# Sales Dashboard - Power BI

## Overview
This project is an interactive Sales Dashboard created in Power BI to analyze sales performance, profit, discounts, and quantity sold. The dashboard provides valuable business insights through visualizations and KPI cards.

## Features
- Total Sales Analysis
- Total Profit Tracking
- Average Discount Monitoring
- Total Quantity Sold
- Sales by Category
- Sales by Sub-Category
- Daily and Monthly Sales Trends
- Customer-wise Sales Analysis
- Dynamic Filters for Better Insights

## Tools Used
- Power BI
- Microsoft Excel

## Dashboard Preview
![Sales Dashboard](screenshot.png)

## Insights
- Technology category generated the highest sales.
- Phones and Chairs were among the top-performing sub-categories.
- Sales trends varied significantly across different days and months.
- Customer-level analysis helps identify high-value customers.

## Purpose
The purpose of this project is to practice data visualization, business analysis, and dashboard design using Power BI.
